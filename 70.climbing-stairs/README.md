# 70. 爬楼梯 (Climbing Stairs)

## 题目描述

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 `1` 阶或 `2` 阶。你有多少种不同的方法可以爬到楼顶呢？

**示例 1：**

```text
输入：n = 2
输出：2
解释：有两种方法可以爬到楼顶。
1. 1 阶 + 1 阶
2. 2 阶
```

**示例 2：**

```text
输入：n = 3
输出：3
解释：有三种方法可以爬到楼顶。
1. 1 阶 + 1 阶 + 1 阶
2. 1 阶 + 2 阶
3. 2 阶 + 1 阶
```

**提示：**

- `1 <= n <= 45`

## 算法知识

### 动态规划

这题是动态规划里的经典入门题。

关键在于想清楚：

> 到第 `i` 阶的方法数，是怎么由前面的台阶推出来的？

因为每次只能爬 `1` 阶或 `2` 阶，所以到第 `i` 阶时，最后一步只可能是：

1. 从第 `i - 1` 阶爬 1 阶上来
2. 从第 `i - 2` 阶爬 2 阶上来

所以到第 `i` 阶的方法数，就是这两种来源的方法数之和。

## 解法：动态规划（推荐）

### 思路

定义：

```text
dp[i] = 爬到第 i 阶的方法数
```

那么根据上面的分析，可以得到状态转移：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

因为：

- 走到第 `i - 1` 阶的所有方法，最后再爬 1 阶，就能到第 `i` 阶
- 走到第 `i - 2` 阶的所有方法，最后再爬 2 阶，也能到第 `i` 阶

这两部分加起来就是总方法数。

### 初始条件

先看最小的几种情况：

- `n = 1` 时，只有 1 种方法
- `n = 2` 时，有 2 种方法

所以：

```text
dp[1] = 1
dp[2] = 2
```

然后从第 3 阶开始一直往后推就可以了。

### 为什么这题和斐波那契数列很像

看转移式：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

这和斐波那契数列的结构完全一样。

所以你也可以把这题理解成：

- 只是初始值不同的斐波那契型动态规划

很多后面的 DP 题，都会有这种“当前状态由前两个状态推出”的味道。

### 以 `n = 5` 为例

我们一步一步推：

```text
dp[1] = 1
dp[2] = 2
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
dp[4] = dp[3] + dp[2] = 3 + 2 = 5
dp[5] = dp[4] + dp[3] = 5 + 3 = 8
```

所以答案是：

```text
8
```

### 复杂度分析

- **时间复杂度：** `O(n)`，从 3 推到 `n`
- **空间复杂度：** `O(n)`，使用一个数组保存状态

## 代码

### C++

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    int climbStairs(int n) {
        if (n <= 2) {
            return n;
        }

        vector<int> dp(n + 1, 0);
        dp[1] = 1;
        dp[2] = 2;

        for (int i = 3; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        return dp[n];
    }
};
```

### Python

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n

        dp = [0] * (n + 1)
        dp[1] = 1
        dp[2] = 2

        for i in range(3, n + 1):
            dp[i] = dp[i - 1] + dp[i - 2]

        return dp[n]
```

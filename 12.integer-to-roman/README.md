# 12. 整数转罗马数字 (Integer to Roman)

## 题目描述

罗马数字包含以下七种字符：

```text
I   表示 1
V   表示 5
X   表示 10
L   表示 50
C   表示 100
D   表示 500
M   表示 1000
```

例如：

- `2` 写作 `II`
- `12` 写作 `XII`
- `27` 写作 `XXVII`

通常情况下，罗马数字中小的数字在大的数字右边。但也存在特殊的减法规则：

- `4` 写作 `IV`，而不是 `IIII`
- `9` 写作 `IX`
- `40` 写作 `XL`
- `90` 写作 `XC`
- `400` 写作 `CD`
- `900` 写作 `CM`

给你一个整数 `num`，请将其转换为罗马数字。

**示例 1：**

```text
输入：num = 3
输出："III"
```

**示例 2：**

```text
输入：num = 58
输出："LVIII"
解释：50 = L，5 = V，3 = III。
```

**示例 3：**

```text
输入：num = 1994
输出："MCMXCIV"
解释：1000 = M，900 = CM，90 = XC，4 = IV。
```

**提示：**

- `1 <= num <= 3999`

## 算法知识

### 贪心算法

贪心算法的核心思想是：每一步都优先选择当前看起来最优的方案，并希望通过局部最优得到全局最优。

本题中，我们希望把整数尽快拆成若干个合法的罗马数字片段。一个自然的想法是：

- 每次都优先使用不超过当前数字的最大罗马数值

这样可以最快减少剩余数字，并且始终符合罗马数字的书写规则。

### 罗马数字的特殊规则

本题不能只考虑 `1, 5, 10, 50, 100, 500, 1000` 这些基本字符，还必须把减法形式也加入候选表中：

- `4 -> IV`
- `9 -> IX`
- `40 -> XL`
- `90 -> XC`
- `400 -> CD`
- `900 -> CM`

只要把这些特殊情况提前放进候选列表，整个转换过程就会变得非常统一，不需要单独写很多分支判断。

## 解法：贪心枚举（推荐）

### 思路

准备一个按数值从大到小排列的映射表：

```text
1000 -> M
900  -> CM
500  -> D
400  -> CD
100  -> C
90   -> XC
50   -> L
40   -> XL
10   -> X
9    -> IX
5    -> V
4    -> IV
1    -> I
```

然后从前往后遍历这张表：

1. 如果当前数值 `value` 小于等于 `num`
2. 就不断使用这个罗马数字，把对应字符拼接到答案后面
3. 同时让 `num -= value`
4. 直到 `num < value`，再看下一个更小的数值

因为我们总是优先选取当前可用的最大数值，所以最终得到的就是合法且正确的罗马数字表示。

### 为什么贪心可行？

因为罗马数字的标准写法本身就是按从大到小组合，并且特殊减法规则也已经被提前编码进候选表中。

例如 `1994`：

1. 先取 `1000 -> M`，剩下 `994`
2. 再取 `900 -> CM`，剩下 `94`
3. 再取 `90 -> XC`，剩下 `4`
4. 最后取 `4 -> IV`

得到结果：`MCMXCIV`

整个过程不需要回退，也不会错过更优写法，因此贪心是成立的。

### 复杂度分析

- **时间复杂度**：O(1)，候选数值只有固定的 13 个，最多循环常数次。
- **空间复杂度**：O(1)，除结果字符串外只使用常数额外空间。

## 代码

### C++

```cpp
#include <vector>
#include <string>
using namespace std;

class Solution {
public:
    string intToRoman(int num) {
        vector<pair<int, string>> symbols = {
            {1000, "M"}, {900, "CM"}, {500, "D"}, {400, "CD"},
            {100, "C"},  {90, "XC"},  {50, "L"},  {40, "XL"},
            {10, "X"},   {9, "IX"},   {5, "V"},   {4, "IV"},
            {1, "I"}
        };

        string ans;
        for (auto& [value, symbol] : symbols) {
            while (num >= value) {
                ans += symbol;
                num -= value;
            }
        }

        return ans;
    }
};
```

### Python

```python
class Solution:
    def intToRoman(self, num: int) -> str:
        symbols = [
            (1000, "M"), (900, "CM"), (500, "D"), (400, "CD"),
            (100, "C"), (90, "XC"), (50, "L"), (40, "XL"),
            (10, "X"), (9, "IX"), (5, "V"), (4, "IV"),
            (1, "I"),
        ]

        ans = []
        for value, symbol in symbols:
            while num >= value:
                ans.append(symbol)
                num -= value

        return "".join(ans)
```
